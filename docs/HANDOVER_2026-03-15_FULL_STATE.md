## 1. What Is NULLA

A decentralized AI agent. Runs locally (Ollama + qwen2.5:7b), connects to a mesh of meet nodes, shares research and knowledge via Brain Hive. Exposed to users through OpenClaw (web UI) or CLI.

Goal: be a genuinely useful local AI that can also do distributed work across a swarm.

## 3. How To Start NULLA Locally

### All-in-one (recommended)

```bash
bash OpenClaw_NULLA.sh
```

Does: start NULLA API → register OpenClaw agent → start OpenClaw gateway → open browser.

### Manual

**Step 1: Start NULLA API server**
```bash
env NULLA_OLLAMA_MODEL=qwen2.5:7b python3 -m apps.nulla_api_server --bind 127.0.0.1 --port 11435
```

**Step 2: Verify**
```bash
curl -ksS http://127.0.0.1:11435/healthz
```

**Step 3: Start OpenClaw gateway**
```bash
~/.nvm/versions/node/v22.22.1/bin/node ~/.nvm/versions/node/v20.20.0/lib/node_modules/openclaw/openclaw.mjs gateway run --force
```

**Step 4: Open in browser**
```
http://127.0.0.1:18789/#token=<your-token>
```

**Trace Rail** (runtime events):
```
http://127.0.0.1:11435/trace
```

### Alt ports (SSH forwarding detected)

If SSH port forwarding is detected, ports shift:
- API: 11435 → 21435
- Gateway: 18789 → 28790

## 8. What's Broken — THE MAIN PROBLEM

**The plumbing works. The model output quality is the blocker.**

### P0: Model Response Quality ("Final Tuning")

1. **"couldn't produce a clean final summary"** — Model polish step fails; shows raw fallback. qwen2.5:7b often returns generic planner text or times out.

2. **Describe-not-execute loop** — User says "proceed"/"do it"/"carry on" and NULLA repeats the identical plan 5+ times instead of executing. The model doesn't emit tool intents.

3. **Shallow research** — Produces generic advice ("use VS Code"), not real web evidence. Research query quality and grounding need improvement.

4. **Persona not sticky** — Character override from chat doesn't persist.

5. **Factual hallucination** — Gets simple facts wrong (Mert/Helius, Toly/Solana). Should force web lookup for entity questions.

### P1: Known Alpha Issues

- Builder-gap classification wrong (1 failing test)
- Time followup weak outside Vilnius (Roma, London)
- Current-events escalation weak
- Named-entity research weak (fuzzy retry needed)
- Simple fact sanity weak
- Full Hive task completion not proven end-to-end

## 13. Non-Negotiable Rules

1. Never conflate watcher board state with public research-packet state.
2. Never call Hive empty without checking watcher dashboard.
3. Never claim local NULLA is live without passing `healthz`.
4. When user says "do it" / "proceed" / "carry on" — EXECUTE, don't describe again.
5. Never paste secrets or tokens.
6. If a surface is down, say so. Don't guess.
7. Check server logs, not just chat output.
8. Owner types fast with typos — interpret intent, don't be pedantic.
9. Don't add requirements the user didn't mention (anti-hallucination).
